# 제안서 작성 (81-proposal-writer)

> MoAI-Cowork v0.2.0 하네스 레퍼런스

## 개요

클라이언트 요구사항 분석, 제안 개발, 제안서 작성, 검증을 수행하는 에이전트 팀 협업 하네스입니다.

## 전문가 역할

- **요구사항 분석가**: 클라이언트 요구사항을 분석합니다.
  - 목표: 클라이언트의 목표 파악
  - 문제: 해결할 문제 파악
  - 제약: 제약 조건 파악
  - 예산: 예산 범위 파악
  - 기한: 프로젝트 기한

- **제안 전략가**: 맞춤형 제안을 개발합니다.
  - 솔루션: 문제 해결 방안 개발
  - 차별성: 경쟁 우위 강조
  - 가치: 클라이언트가 얻을 가치
  - 타이밍: 구현 일정
  - 가격: 가격 책정

- **제안서 작성자**: 전문적인 제안서를 작성합니다.
  - 구조: 논리적 구조 설계
  - 콘텐츠: 각 섹션별 내용 작성
  - 비주얼: 효과적인 시각화
  - 토폴로지: 명확하고 설득력 있는 표현
  - 검증: 논리의 일관성 검증

- **리뷰어**: 제안서의 품질을 검증합니다.
  - 완전성: 모든 필수 요소 포함 확인
  - 정확성: 주장의 정확성 확인
  - 명확성: 표현의 명확성 확인
  - 설득력: 설득력 있는 논거 확인
  - 오류: 문법 및 오류 검증

## 워크플로우

### 단계 1: 준비

1. 사용자 입력에서 추출:
   - **클라이언트**: 클라이언트 정보
   - **요구사항**: RFP 또는 요구사항
   - **솔루션**: 제안할 솔루션
   - **예산**: 가격 정보
   - **기한**: 제안 제출 기한

2. `_workspace/` 디렉토리 생성

### 단계 2: 팀 구성 및 실행

| 순서 | 작업 | 담당 | 의존성 | 산출물 |
|------|------|------|--------|--------|
| 1 | 분석 | 분석가 | 없음 | `01_requirements.md` |
| 2 | 전략 | 전략가 | 작업 1 | `02_proposal_strategy.md` |
| 3 | 작성 | 작성자 | 작업 2 | `03_proposal_document.md` |
| 4 | 검증 | 리뷰어 | 작업 3 | `04_review_report.md` |

## 산출물

- `01_requirements.md` — 요구사항 분석
- `02_proposal_strategy.md` — 제안 전략
- `03_proposal_document.md` — 제안서
- `04_review_report.md` — 검증 보고서

## 오류 처리

| 오류 유형 | 대응 전략 |
|---------|---------|
| 요구사항 불명확 | 클라이언트 재문의 권고 |
| 제안 불충분 | 추가 콘텐츠 보강 |
| 에이전트 실패 | 1회 재시도 → 진행 |
